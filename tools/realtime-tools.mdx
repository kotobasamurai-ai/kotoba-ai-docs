リアルタイムツールは、通話中にエージェントがその場で呼び出すツールです。通話終了、転送、SMS送信など、会話の進行に直接関わる処理を有効化します。

エージェント設定の「リアルタイムツール」で有効化します。必要なものだけをオンにしてください。

## 設定のポイント

- 会話の流れを変える処理だけを有効にする
- 有効化したツールは、プロンプト側で使う条件を明記する
- 通話後でよい処理は、[通話後ツール](/tools/post-call-tools)へ分ける

---

## 通話終了（end_session）

エージェントが通話を終了できるようにするツールです。

### 活用シーン

- 対応が完了し、終話してよいと判断できたとき
- 終話の挨拶を済ませ、通話を閉じたいとき

### 有効化の方法

「リアルタイムツール」で **end_session** をオンにします。あわせて、どの条件で終話するかをプロンプトに記述してください。

**プロンプト記述例**

> 対応が完了したら、以下の手順で終話してください。
>
> 1. 「他にご不明な点はございますか」と確認し、回答を待ちます（最大2回まで。2回応答があれば内容に関わらず終話へ進む）
> 2. 不明点がない場合 → 「本日は○○サポートセンターのAIオペレーターが承りました。お電話ありがとうございました。失礼いたします。」と伝え、直後に end_session を呼んでください（ユーザーの返答を待たない）
> 3. 不明点がある場合 → 対応してから1に戻る

<Warning>必ず終話の挨拶を先に発話してから終了させてください。</Warning>

<Warning>ユーザーが話してる途中で終了しないよう、プロンプトに条件を明示してください。</Warning>

## オペレーター転送（handoff_to_human）

エージェントがオペレーターへ通話を転送するツールです。

### 活用シーン

このツールは、エージェントが**意図的に人間へつなぐ**場面で使います。

- ユーザーが「担当者につないでほしい」と希望したとき
- 契約内容の変更など、業務上の判断が必要なとき
- VIP対応や特定の用件を有人で処理したいとき

<Info>
  クレームや強い不満など、AIでは対応が難しい場合の自動エスカレーションは、エージェント設定の「エスカレーション条件」とデフォルト転送先の組み合わせで実現します。
</Info>

### 有効化の方法

「リアルタイムツール」で **handoff_to_human** をオンにします。設定後、ツール説明に転送条件と転送先を書きます。

**ツール説明の記述例**

> ユーザーが担当者への接続を希望した場合、または以下の用件に該当する場合に転送してください。
>
> `builtin_tool(command="handoff_to_human" destination="sip:sales@pbx.example.com")`
>
> 転送先一覧:
> - 営業部: sip:sales@pbx.example.com
> - サポート: sip:support@pbx.example.com

### 入力パラメータ

| パラメータ | 説明 |
| --- | --- |
| destination | 転送先（SIP URI または電話番号） |
| reason | 転送理由 |
| briefing | 引き継ぎメモ（転送先のオペレーターへの申し送り） |

<Tip>
  用件ごとに転送先が異なる場合は、転送先の一覧と選択条件を説明欄にまとめておくと運用しやすくなります。
</Tip>

## 対応手順取得（get_instruction）

用件ごとの対応手順を、必要なときに参照させるツールです。キーを指定して呼び出すと、対応する手順が返ります。

### 活用シーン

- 用件ごとに異なる対応手順がある場合
- 手順が長く、プロンプトに直接書くと肥大化する場合
- 複数の業務フローを一つのエージェントにまとめたい場合

### 有効化の方法

1. 「リアルタイムツール」セクションで **get_instruction** のスイッチをオンにします
2. 「ツール説明」にエージェントがこのツールをいつ呼ぶかを記述します
3. 「エントリーを追加」でキーと手順を登録します

**ツール説明の記述例**

> ユーザーの用件が判明したら、対応するキーを指定してこのツールを呼んでください。

### エントリーの設定

| 項目 | 説明 | 例 |
| --- | --- | --- |
| キー | 用件の識別名。ツール呼び出し時に指定します | `契約変更`、`解約申請`、`故障相談` |
| 対応手順 | エージェントに返す手順本文です | 「お名前と電話番号を確認してから…」 |

**設定例**

キー: `故障相談`

対応手順:
> まず製品の型番を確認してください。型番が判明したら、購入年月を確認します。保証期間内（購入から1年）であれば無償修理を案内し、期間外であれば有償修理の見積もりを案内してください。

## 電話番号バリデーション（validate_phone_number）

日本の電話番号を検証し、整形するツールです。

### 活用シーン

- 発信者や折り返し先の電話番号を聞き取ったとき
- 復唱確認の前に桁数・形式を確認したいとき
- 聞き取り不備や桁不足を早めに検出したいとき

### 有効化の方法

「リアルタイムツール」で **validate_phone_number** をオンにします。電話番号を聞き取った時点で呼ぶよう、プロンプトに条件を書いてください。

**プロンプト記述例**

> 電話番号を聞き取ったら validate_phone_number を呼び、ai_instruction に従って復唱確認してください。

### 入力パラメータ

| パラメータ | 説明 | 例 |
| --- | --- | --- |
| phone_number | 電話番号（ハイフンあり/なし、全角可） | `09012345678`、`090-1234-5678` |

### 主な戻り値

| フィールド | 型 | 説明 |
| --- | --- | --- |
| success | boolean | バリデーション成功かどうか |
| formatted | string | ハイフン区切りの正規形式（例: `090-1234-5678`） |
| partial | boolean | 部分入力かどうか（7桁以下の場合 true） |
| ai_instruction | string | エージェントへの次のアクション指示 |
| error | string | エラーメッセージ（失敗時のみ） |

### 動作の詳細

- 全角数字を半角に変換します
- ハイフン・括弧・長音記号などを除去します
- 携帯電話・固定電話・フリーダイヤル・ナビダイヤルなどを判定します
- **7桁以下**は部分入力とみなし `partial: true` を返します。`ai_instruction` に再聴取の案内が含まれます
- **バリデーション成功時**は `ai_instruction` に復唱確認の指示が含まれます

<Tip>
  電話番号の復唱や再確認の流れを揃えたい場合は、`ai_instruction` を参照する前提でプロンプトを書くと安定します。
</Tip>

## SMS 送信（send_sms）

指定した電話番号へSMSを送信するツールです。

### 活用シーン

- 手続きページや確認フォームの URL を送りたいとき
- 折り返し番号や受付番号を文字で伝えたいとき

### 有効化の方法

「リアルタイムツール」で **send_sms** をオンにします。

有効化すると「**モックモード（実際に SMS を送信しない）**」のトグルが表示されます。テスト中は必ずオンにし、本番運用時はオフにしてください。

### 入力パラメータ

| パラメータ | 説明 | 例 |
| --- | --- | --- |
| mobilenumber | 送信先電話番号（数字のみ・11桁） | `09012345678` |
| smstext | SMS 本文（最大 140 バイト） | `ご確認用 URL をお送りします: https://...` |

### 主な戻り値

| フィールド | 型 | 説明 |
| --- | --- | --- |
| success | boolean | 送信成功かどうか |
| status_code | integer | HTTP レスポンスコード（200 = 成功） |
| error | string | エラーメッセージ（失敗時のみ） |

<Warning>電話番号はハイフンなしの数字のみで渡します。</Warning>

<Warning>本文は140バイト以内に収めてください。超過した場合は送信に失敗します。</Warning>

<Tip>送信後は音声でも完了を案内するとユーザーに伝わりやすくなります。</Tip>

## フォーム送信（send_form）

フォームURLをSMSで送信するツールです。口頭では正確に聞き取れない情報をフォーム入力に切り替えたい場合に使います。

### 活用シーン

- 住所・メールアドレス・氏名・製品型番など、口頭では聞き取りにくい情報を収集したいとき（誤字や聞き間違いが起きやすい項目に特に有効）
- 同意確認や申し込みフォームをユーザーに記入してもらいたいとき
- 決済リンクを送りたいとき

### 有効化の方法

「リアルタイムツール」で **send_form** をオンにします。有効化するとフォーム定義を設定できます。

**フォーム定義の項目**

| 項目 | 説明 |
| --- | --- |
| タイトル | ユーザーに表示されるフォーム名 |
| SMS メッセージ | 送信時の本文。`{{ form_url }}` がフォーム URL に自動で置換されます |
| フィールド | テンプレート内の入力項目を選択します |

### 入力パラメータ

| パラメータ | 説明 | 例 |
| --- | --- | --- |
| form_id | 送信するフォームの ID（フォーム定義で設定したキー） | `address_form` |
| phone_number | SMS 送信先の電話番号（数字のみ） | `09012345678` |
| product | 決済対象の商品 ID（決済が不要な場合は省略） | — |

### 主な戻り値

| フィールド | 型 | 説明 |
| --- | --- | --- |
| success | boolean | SMS 送信が成功したかどうか |
| status | string | `sms_sent`（送信完了） |
| form_url | string | 送信したフォームの URL |
| error | string | エラーメッセージ（失敗時のみ） |

<Info>
  フォーム送信後は、ユーザーに「SMSのリンクから入力してください」と口頭でも案内しておくと完了率が上がります。
</Info>

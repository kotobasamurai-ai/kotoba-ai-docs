ツールは、エージェントが会話中や通話後に呼び出す外部処理です。Python でカスタムロジックを記述し、エージェントの `実行` コマンドから利用できます。

<Info>
  ツールの**呼び出し方**（Kotobaフローでの `実行` コマンド）については [ツール連携](/agents/tools) を参照してください。このページではツールの**作り方**を説明します。
</Info>

## ツール一覧

サイドバーの「ツール」をクリックすると、組織に登録されたツールの一覧が表示されます。

| 操作 | 説明 |
|------|------|
| 検索 | ツール名・説明文でフィルタリング |
| 新規作成 | テーブル上部の「新しいツールを作成...」をクリック |
| 複製 | 既存ツールをコピーして新しいツールを作成 |
| 削除 | 不要なツールを削除（確認ダイアログが表示されます） |

## ツールの作成・編集

ツールの編集画面には **設定** タブと **コード** タブがあります。

### 設定タブ

ツールの基本情報と入出力を定義します。

#### 基本情報

| 項目 | 説明 |
|------|------|
| ツール名 | ヘッダーのツール名をクリックしてインライン編集できます |
| 説明 | ツールの目的を記述します。AIがツールを選択する際の判断材料になります |

#### 入力パラメータ（inputs）

ツールが受け取るパラメータを定義します。

| プロパティ | 説明 |
|-----------|------|
| パラメータ名 | 入力値の名前 |
| 型 | `string`, `number`, `boolean` など |
| 説明 | パラメータの用途（AIがパラメータを埋める際の手がかり） |
| 必須 | 必須パラメータかどうか |

#### 戻り値（returns）

ツールが返す値のスキーマを定義します。Kotobaフロー内で `{{ @ツール名.プロパティ }}` として参照されます。

### コードタブ

Python コードエディタでツールのロジックを記述します。

```python
def main(input1: str, input2: int) -> dict:
    # ここにツールのロジックを記述
    result = process(input1, input2)
    return {"status": "success", "data": result}
```

<Tip>
  **実行ボタン** でコードをサンドボックス環境でテスト実行できます。実行結果（stdout / stderr / エラー）と実行時間がエディタ下部に表示されます。
</Tip>

## テスト実行

コードタブの **実行** ボタンをクリックすると、E2B サンドボックス環境でコードがテスト実行されます。

- **stdout**（標準出力）: 白色で表示
- **stderr**（標準エラー）: 黄色で表示
- **エラー**: 赤色で表示
- 実行時間がミリ秒単位で表示されます

## 保存と公開

| 操作 | 説明 |
|------|------|
| **保存** | 編集内容を保存します（下書き状態） |
| **公開** | 保存した上でバージョン付きスナップショットを作成します。エージェントから参照されるのは公開済みバージョンです |

<Info>
  エージェントのルーティングでバージョンを固定している場合、ツールを更新しても公開するまでエージェントには反映されません。
</Info>

## バージョン管理

公開するたびにバージョン番号が自動的に付与されます。

- **バージョン履歴**: ヘッダーのバージョン履歴ボタンから過去のバージョン一覧を確認できます
- **復元**: 任意の過去バージョンを選択して復元できます（コード・設定・入出力すべてが復元されます）

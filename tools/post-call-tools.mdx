通話後ツールは、通話が終了した直後に自動実行されるツールです。要約生成、情報抽出、CRM登録など、会話後にまとめて処理したい作業を自動化できます。

エージェント設定の「通話後ツール」で管理します。

## 追加方法

標準ツールはスイッチで有効化します。カスタムツールは「ツールを追加」から選択して追加します。

## 設定のポイント

- ツールはリストの順に実行される。前のツールの戻り値を次のツールで参照できる
- 通話中に呼ぶ必要がない処理は、できるだけここに寄せる

---

## 通話要約生成（generate_call_summary）

通話内容から構造化された要約を生成するツールです。

### 活用シーン

- 通話後に要約を自動生成してセッション画面で確認したいとき
- 後続の CRM 登録ツールに要約を渡したいとき
- 分析基盤に通話カテゴリや感情情報を送りたいとき

### 入力パラメータ

ツール名をクリックして開くダイアログから設定します。

| パラメータ | 説明 |
| --- | --- |
| transcript | 要約対象の会話履歴 |
| language | 要約の言語（デフォルト: 日本語） |

### 主な戻り値

| フィールド | 型 | 説明 |
| --- | --- | --- |
| title | string | 通話の短いタイトル（20文字以内） |
| summary | string | 通話内容の要約（200文字以内） |
| action_items | array | フォローアップが必要なアクションのリスト |
| category | string | 問い合わせカテゴリ（問い合わせ / 苦情 / 予約 / その他） |
| sentiment | string | 顧客の感情（positive / neutral / negative） |

<Tip>
  `generate_call_summary` を先頭に置くと、後続のカスタムツールで要約結果を利用しやすくなります。
</Tip>

## 変数抽出（extract_variables）

通話の録音または文字起こしから、必要な項目を抽出するツールです。

### 活用シーン

- 通話中に聞き取った情報（名前・住所・型番など）を後から構造化して取り出したいとき
- 音声録音を使って聞き取り精度を上げたいとき
- 抽出した情報を CRM や業務システムに登録したいとき

### 入力パラメータ

| パラメータ | 必須 | 説明 |
| --- | --- | --- |
| fields | 必須 | 抽出するフィールド名と説明の辞書 |
| transcript | どちらか必須 | 文字起こしテキスト |
| storage_path | どちらか必須 | 録音ファイルのストレージパス |
| prompt | 任意 | 抽出指示プロンプト（デフォルトあり） |

`storage_path` を指定すると録音も参照して抽出します。`transcript` のみでも利用できます。

### 戻り値

指定したフィールド名をキーにした抽出結果が返ります。`storage_path` を指定した場合は、関連する録音URLも返されます。

### フィールド設定例

抽出したいフィールド名と説明を設定します。説明は具体的に書くほど安定します。

| フィールド名 | 説明 |
| --- | --- |
| 氏名 | 発信者のフルネーム |
| 電話番号 | 折り返し先の電話番号 |
| 問い合わせ内容 | 主な問い合わせや依頼内容 |
| 型番 | 製品の型番（言及された場合） |

<Info>
  抽出結果を後続のカスタムツールに渡す場合は、フィールド名を業務システム側の項目名に近づけておくと扱いやすくなります。
</Info>

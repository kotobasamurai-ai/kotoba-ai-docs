カスタムツールは、Python で独自ロジックを実装できるツールです。顧客検索、CRM更新、社内システム連携など、標準ツールでは足りない処理を追加したい場合に使います。

作成したツールは公開後、エージェントのリアルタイムツールまたは通話後ツールとして追加できます。

## ツール一覧

「ツール」画面では、組織内で作成したカスタムツールを管理します。

| 操作 | 説明 |
| --- | --- |
| 検索 | ツール名や説明で絞り込みます |
| 新規作成 | 「新しいツールを作成...」から新規作成します |
| 複製 | 既存ツールをコピーして別のツールとして編集します |
| 削除 | 不要なツールを削除します |

## ツールの作成・編集

編集画面は「**設定**」タブと「**コード**」タブに分かれています。

### 設定タブ

ツールの基本情報と入出力を定義します。

#### 基本情報

| 項目 | 説明 |
| --- | --- |
| ツール名 | 管理画面やエージェント設定で表示される名前です |
| 説明 | ツールの目的と利用条件を記述します。AIがツール選択時の手がかりとして参照します |

#### 入力パラメータ（inputs）

ツールが受け取るパラメータを定義します。

| プロパティ | 説明 |
| --- | --- |
| パラメータ名 | 入力値のキー名です |
| 型 | `string`、`number`、`boolean` などを指定します |
| 説明 | 値の意味や期待する形式を記述します |
| 必須 | 必須入力かどうかを指定します |

#### 戻り値（returns）

ツールが返す値のスキーマを定義します。通話後ツールとして連続実行する場合、後続ツールからこの戻り値を参照できます。

### コードタブ

Python コードエディタでツールのロジックを記述します。

```python
def main(input1: str, input2: int) -> dict:
    # ここにツールのロジックを記述
    result = process(input1, input2)
    return {"status": "success", "data": result}
```

<Tip>
  説明文、入力スキーマ、戻り値スキーマを先に整えておくと、エージェント側での利用条件が明確になります。
</Tip>

## テスト実行

コードタブの **実行** で、サンドボックス環境でのテスト実行ができます。

- `stdout`: 標準出力
- `stderr`: 標準エラー
- `error`: 実行エラー
- 実行時間: ミリ秒単位で表示

## 保存と公開

| 操作 | 説明 |
| --- | --- |
| **保存** | 編集内容を下書きとして保存します |
| **公開** | 保存済みの内容を公開版として確定します。エージェントが参照するのは公開版です |

<Info>
  ツールを更新しても公開するまでエージェントには反映されません。
</Info>

## バージョン管理

公開のたびにバージョンが作成されます。

- **バージョン履歴**: 過去の公開版を確認できます
- **復元**: 任意の版に戻せます

## エージェントへの追加

カスタムツールを公開したら、エージェント設定の「**リアルタイムツール**」または「**通話後ツール**」で「ツールを追加」から選択します。

通話中に即時実行したい処理はリアルタイムツールへ、記録や抽出など通話後にまとめて処理できるものは通話後ツールへ追加してください。

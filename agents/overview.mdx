エージェント設定では、電話で応対するAIの名前、会話方針、利用ツール、音声、モデル、転送条件をまとめて管理します。

## 使い方の流れ

エージェント設定は、次の順序で進めると整理しやすくなります。

1. 「基本情報」でエージェント名、オープニングメッセージ、プロンプトを設定する
2. 「通話中ツール」と「通話後ツール」で必要な機能だけを有効にする
3. 「音声・認識設定」「AIモデル」「通話制御」で応答品質とエスカレーション条件を調整する
4. テスト通話で想定どおりに動くか確認する
5. 問題がなければ「公開」を実行する
6. 公開後に[ルーティング](/routing/overview)で電話番号や着信先に紐づける

<Info>
  入力内容は自動保存されますが、本番環境に反映されるのは「公開」後です。
  保存だけでは利用中のバージョンは切り替わりません。
</Info>

## 画面上部の操作

設定画面の上部には、編集中によく使う操作がまとまっています。

| 項目 | 説明 |
| --- | --- |
| 戻る | エージェント一覧に戻ります |
| エージェント名 | 上部バーからも直接編集できます |
| 保存状態 | 変更内容は自動保存されます。保存は下書きとして扱われます |
| バージョン履歴 | 過去に公開した版を確認し、必要に応じて戻せます |
| テスト通話 | 現在の設定で通話テストを行います |
| 公開 | 保存済みの内容を検証し、新しい公開版として反映します |

<Tip>
  テスト通話を始めるには、オープニングメッセージの入力が必要です。
</Tip>

## 基本情報

### エージェント名

管理用の名前です。エージェント一覧や運用画面に表示されます。用途がわかる名前にすると、検索や保守が容易になります。

同じ組織内で重複する名前は保存できません。

**例**: `受付対応エージェント`、`予約確認エージェント`

### タグ

ダッシュボード上の検索や整理に使うラベルです。エージェントの動作には直接影響しません。

### オープニングメッセージ

通話開始直後に固定文として再生されるメッセージです。入力した文面がそのまま音声として読み上げられます。

そのため、次のような用途に適しています。

- 会社名や担当名の名乗り
- 受付の定型案内
- 本人確認や録音案内の冒頭文

### プロンプト

エージェントの振る舞いを決める中心設定です。以下の内容を含めることを推奨します。

- **役割**（例: カスタマーサポート担当）
- **話し方のトーン**（例: です・ます調の敬語）
- **対応範囲**（例: 製品に関する質問のみ対応）
- **禁止事項**（例: 個人情報の開示禁止）
- **ツールの利用条件**
- **転送やエスカレーションの判断基準**

実務では、次のように役割を分けると安定します。

- オープニングメッセージ: 固定の挨拶や名乗り
- プロンプト: 何を達成するか、何を確認するか、どの条件で何を行うか

<Tip>
  プロンプトは具体的に書くほどAIの応答が安定します。「丁寧に対応して」よりも「です・ます調の敬語で、相手の名前を呼びながら対応して」のように記述しましょう。
</Tip>

## 通話中ツール

通話中ツールは、会話の途中でAIがその場で呼び出せる機能です。顧客検索、予約照会、SMS送信など、会話の進行に必要な処理をここで有効にします。

### カスタムツール

「ツールを追加」から既存のツールを選択して追加します。追加したツールは、右側のボタンで一時的に無効化したり、削除したりできます。

### 組み込みツール

標準で用意されている代表的な機能です。

| ツール | 説明 |
| --- | --- |
| `end_session` | AIが自分で通話を終了できるようにします |
| `handoff_to_human` | オペレーターや担当者へ転送できるようにします |
| `get_instruction` | 用件に応じた対応手順を参照できるようにします |

### プラットフォームツール

`send_sms` など、プラットフォーム標準のツールを有効化できます。必要なものだけを選択してください。

<Tip>
  会話の途中で呼ぶ必要がある処理だけを通話中ツールに置くと、プロンプトを簡潔に保ちやすくなります。
</Tip>

## フォーム定義

`send_form` を有効にすると、フォーム定義セクションが表示されます。

フォームは、口頭では回収しづらい情報をSMS経由で入力してもらいたい場合に便利です。

| 項目 | 説明 |
| --- | --- |
| フォーム追加 | テンプレートから作成します |
| タイトル | ユーザーに表示されるフォーム名 |
| SMSメッセージ | フォーム送信時の文面。`{{ form_url }}` はフォームURLに置換されます |
| フィールド | テンプレート内の項目をON/OFFできます。必須項目も表示されます |

## 通話後ツール

通話後ツールは、通話終了後に自動実行される処理です。会話中に使う必要がない後処理は、こちらにまとめると設計しやすくなります。

代表的な用途:

- 通話要約の生成
- CRMや業務システムへの記録
- 必要情報の抽出
- ログや分析基盤への連携

<Info>
  通話後ツールとして利用できるのは、あらかじめ[ツール管理](/tools/overview)で設定済みのツールです。
</Info>

## 音声・認識設定

### 音声（TTS）

| パラメータ | 範囲 | デフォルト | 説明 |
| --- | --- | --- | --- |
| 音声 | 女性 / 男性 | 女性 | エージェントの音声の種類 |
| 話速 | 0.5〜2.0 | 1.0 | 発話のスピード |
| 音量補正 | -10〜+10 dB | 0 | 音量の調整。プラスで大きく、マイナスで小さく |

<Tip>
  デフォルトの話速 1.0 はやや速く感じることがあります。ゆっくり話させたい場合は 0.8〜0.9 に下げるのがおすすめです。
</Tip>

### 音声認識コンテキスト

固有名詞、ブランド名、社内用語、型番、略語などを登録し、音声認識の精度を高めます。

| 項目 | 用途 | 例 |
| --- | --- | --- |
| 単語リスト | よく出る固有名詞を列挙 | `Kotoba`、`プレミアムプラン` |
| キーと値のペア | 単語に補足情報を付与 | `AI Contact`: `AIコンタクト` |
| 背景説明 | 業務内容や文脈を自然文で記述 | 「当社はクラウドPBXを提供しており…」 |

### VAD設定（音声区間検出）

ユーザーが話しているかどうかをリアルタイムで判定する機能です。

| パラメータ | 範囲 | デフォルト | 説明 |
| --- | --- | --- | --- |
| 感度 | 1.0〜15.0 | 6.0 | 高いほど小さな声も拾いやすくなるが、ノイズにも反応しやすくなる |
| 音声保持時間 | 0.0〜1.0秒 | 0.1秒 | 音声が途切れた後も「発話中」とみなす時間 |
| 最小音声長 | 0.0〜0.5秒 | 0.2秒 | この長さ未満の音声は割り込みとして検知しない |

<Tip>
  **AIが早く割り込む場合**: 音声保持時間を長くしてください（例: 0.1 → 0.3）。ユーザーが言葉を選んでいる間の短い沈黙を「発話終了」と判定しにくくなります。

  **雑音で誤反応する場合**: 感度を下げてください（例: 6.0 → 4.0）。また、最小音声長を長くする（例: 0.2 → 0.3）ことで、短いノイズを無視できます。

  **ユーザーの発話を検出できない場合**: 感度を上げてください（例: 6.0 → 8.0）。電話回線の音質が悪い場合に有効です。
</Tip>

### 読み辞書

特定の単語の読み方を補正するための辞書です。社名や製品名など、誤読を避けたい語の統一に使います。

**登録例**

| 単語 | 読み | メモ |
| --- | --- | --- |
| 受注ID | じゅちゅうアイディー | システム固有の単語 |
| IVR | アイブイアール | 電話システムの略語 |
| CRM | シーアールエム | 顧客管理の略語 |

## AIモデル

### 言語モデル

会話生成に使用するモデルです。モデルによって応答速度と精度が異なります。

| モデル | 応答速度（目安） |
| --- | --- |
| Gemini 3.1 Flash Lite | ~0.7秒 |
| Claude Haiku 4.5 | ~0.6秒 |
| Claude Sonnet 4.6 | ~1.0秒 |

<Info>
  応答速度はネットワーク環境や会話の複雑さによって変動します。上記は目安です。
</Info>

まずは軽量モデルで開始し、応答品質が不足する場合のみ高性能モデルへ切り替える運用を推奨します。

### 先読み生成

ユーザーの発話終了を完全に待たず、先に推論を始める設定です。応答は速くなりますが、状況によっては割り込み気味に感じられることがあります。

## 通話制御

通話事故を防ぎ、適切に人へつなぐための設定です。

### デフォルト転送設定

ガードレール発動時や転送先未指定時に使う既定の転送設定です。

| 項目 | 説明 |
| --- | --- |
| SIP転送（Warm） | SIP URI 向けの転送。AIが転送先に引き継ぎメッセージを伝えてから通話者を接続します |
| 外線転送（Cold） | 電話番号向けの転送。通話者を直接転送先に接続します |

| パラメータ | 説明 |
| --- | --- |
| 転送先 | SIP URI（例: `sip:operator@example.com`）または電話番号（例: `+81312345678`） |
| 引き継ぎメッセージ | Warm転送時に転送先へ伝える内容。`auto` を指定すると会話内容から自動生成します |
| 応答待ちタイムアウト | 転送先が応答するまでの待機時間（秒）。タイムアウト時は通話者に転送できなかった旨を伝えます |

### エスカレーション条件

クレーム、強い不満、AIでは処理困難な依頼など、人への切り替えが必要な条件を定義します。

<Info>
  転送先が未設定の場合、エスカレーション条件は利用できません。
</Info>

### 無応答タイムアウト

ユーザーが一定時間応答しない場合の動作を設定します。

| パラメータ | デフォルト | 説明 |
| --- | --- | --- |
| 待機秒数 | 15秒 | ユーザーの応答を待つ時間 |
| 確認メッセージ | — | 無応答時に順番に再生されるメッセージ（複数設定可） |
| 終話メッセージ | — | すべての確認メッセージ再生後、通話終了前に再生されるメッセージ |

#### 動作の流れ

1. ユーザーが**待機秒数**の間応答しない
2. **確認メッセージ**（1つ目）を再生し、再び待機
3. まだ応答がない場合、次の**確認メッセージ**を再生し、再び待機
4. すべての確認メッセージを再生しても応答がない場合、**終話メッセージ**を再生して通話を終了

<Tip>
  確認メッセージは複数設定でき、設定した順番に再生されます。例えば「お客様、聞こえていますでしょうか？」→「お声が聞こえないようですので…」のように段階的に案内できます。

  転送先と無応答時のメッセージは、運用開始前に必ず設定してください。事故防止の効果が高い項目です。
</Tip>

## 詳細設定

### コンテキスト圧縮

長い通話で会話履歴が増えたときに、古い文脈を要約して保持する設定です。長時間のヒアリングやサポート対応が多い場合に有効です。

短時間の電話が中心であれば、初期段階では未設定でも問題ありません。

| パラメータ | デフォルト | 説明 |
| --- | --- | --- |
| 圧縮しきい値 | 10メッセージ | この数を超えると過去の会話が要約されます |
| 圧縮用プロンプト | — | 要約時に使用するカスタムプロンプト |

<Tip>
  圧縮用プロンプトをカスタマイズすると、要約の質を調整できます。例えば「顧客名と問い合わせ内容は必ず保持すること」のように、重要な情報の保持を指示できます。
</Tip>

## 保存と公開

保存と公開は役割が異なります。

| 操作 | 説明 |
| --- | --- |
| 保存 | 下書きとして保存します。編集中の内容は保持されますが、本番利用中のバージョンは切り替わりません |
| 公開 | 検証を行ったうえで新しいバージョンを作成し、そのバージョンを利用中の設定として反映します |

編集しただけでは本番環境には反映されません。実際に利用される設定を切り替えるには、「公開」をクリックする必要があります。

---
title: ツール連携
description: 実行 を使った外部ツールの呼び出し方
icon: wrench
---

`実行`（tool）を使って、会話中に外部ツールを呼び出せます。顧客検索、CRM更新、時刻取得などの処理を会話フローに組み込めます。

## 基本構文

```yaml
- 実行: "@ツール名"
  inputs:
    パラメータ名: "{{ 変数 }}"
```

ツール名の先頭には `@` を付けます。`inputs` でパラメータを渡します。

## ツール結果の参照

ツールの実行結果は `{{ @ツール名.プロパティ }}` で参照できます。

```yaml
- 実行: "@時刻取得"
- 指示: "現在 {{ @時刻取得.hour }} 時であることを伝える"
```

## 実践例：顧客検索

```yaml
- 指示: "顧客IDを聞く"
- 待機: "顧客IDを聞き取るまで"
  saves:
    - 顧客ID:
        type: string
        description: "お客様のID"

- 実行: "@customer_lookup"
  inputs:
    id: "{{ 顧客ID }}"

- if: "{{ @customer_lookup.success }} == False"
  then:
    - 指示: "顧客が見つからないことを伝える"
    - 終了: true
- else:
  then:
    - 指示: "{{ @customer_lookup.name }}様、ご本人確認ができました"
```

## from_llm — AIが文脈から推論する入力

パラメータに `from_llm: true` を指定すると、会話の文脈からAIが値を推論して入力します。ユーザーに明示的に聞かなくても、会話の流れから適切な値を判断します。

```yaml
- 実行: "@customer_lookup"
  inputs:
    顧客ID:
      type: string
      from_llm: true
      description: "会話中に言及された顧客ID"
```

## announcement — 処理中メッセージ

ツール実行に時間がかかる場合、`announcement` で処理中であることをユーザーに伝えられます。

```yaml
- 実行: "@customer_lookup"
  inputs:
    id: "{{ 顧客ID }}"
  announcement: "少々お待ちください、お客様情報を確認しております"
```

## 通話終了後のツール実行（post_call）

通話が終了した後に自動でツールを実行できます。CRM更新や要約生成などに使います。

```yaml
name: "サポートエージェント"
base_prompt: |
  あなたはサポートセンターのオペレーターです。

post_call:
  - 実行: "@generate_summary"
    inputs:
      transcript: "{{ global.transcript }}"

  - 実行: "@crm_update"
    inputs:
      transcript: "{{ global.transcript }}"
      summary: "{{ @generate_summary.summary }}"
      caller_name: "{{ 氏名 }}"

steps:
  - 指示: "挨拶をする"
```

<Info>
  `post_call` 内のツールは上から順に実行されます。前のツール結果を次のツールで参照できます。
</Info>

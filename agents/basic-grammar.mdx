エージェントの会話フローは、**指示**（AIへの指示）と **待機**（ユーザーの回答待ち）の繰り返しで構成されます。

## 会話の基本パターン

すべての会話フローは **指示 → 待機** のループが基本です。

```yaml
steps:
  - 指示: "お名前をフルネームで教えてください"
  - 待機: "お名前を聞き取れるまで"
    saves:
      - 氏名:
          type: string
          description: "お客様の氏名"
  - 指示: "お電話番号を教えてください"
  - 待機: "電話番号を聞き取れるまで"
    saves:
      - 電話番号:
          type: string
          description: "お客様の電話番号"
  - 終了: true
```

<Info>
  **指示** → **待機** のペアを繰り返すことで、会話を1ステップずつ進めていきます。
</Info>

## 指示

AIに「何をすべきか」を自然言語で指示します。AIはこの指示に従って発話を生成します。

```yaml
- 指示: "製品の安全に関する連絡のため、所有者情報を登録することを伝える。名前をフルネームで聞く"
```

<Tip>
  指示の内容は具体的に書くほど、AIの発話が安定します。「名前を聞く」よりも「名前をフルネームで聞き、復唱して確認する」の方が意図通りの会話になります。
</Tip>

### 指示で変数を使う

収集済みの変数は `{{ 変数名 }}` で参照できます。

```yaml
- 指示: "{{ 氏名 }}様、お電話番号は{{ 電話番号 }}でお間違いないですか？"
```

### 連続する指示

**待機** を挟まずに **指示** を連続させると、AIは複数の指示をまとめて実行します。

```yaml
- 指示: "担当サービスショップからの連絡先電話番号を確認する。今かけている番号でよいか聞く"
- 指示: "違う場合は番号を聞き、ハイフン付きで復唱確認する"
- 待機: "ユーザーが復唱確認に同意するまで"
```

## 待機

次のステップに進むための条件を定義します。条件が満たされるまで、AIはユーザーとの会話を続けます。

```yaml
- 待機: "ユーザーが復唱確認に同意するまで"
```

### saves — 情報の収集

**待機** に `saves` を付けると、条件が満たされた時点でユーザーの回答から情報を抽出・保存します。

```yaml
- 待機: "住所を聞き取るか、わからないと回答するまで"
  saves:
    - 住所:
        type: string
        description: "住所。不明の場合は「不明」"
```

### saves のプロパティ

| プロパティ | 型 | デフォルト | 説明 |
|-----------|-----|-----------|------|
| `type` | string | `"string"` | 変数の型。`string`, `number`, `boolean` |
| `description` | string | — | AIが値を抽出する際の手がかり |
| `optional` | boolean | `false` | `true` で値がなくても次に進める |
| `enum` | list | — | 許可する値のリスト |

### enum の例

ユーザーの回答を特定の選択肢に絞りたい場合に使います。

```yaml
saves:
  - 住所同意:
      type: string
      description: "復唱確認に同意するかどうか"
      enum: ["同意", "不同意"]
```

## 変数の参照

`saves` で保存した変数は、後続の **指示** で `{{ 変数名 }}` として参照できます。

```yaml
- 指示: "お名前を聞く"
- 待機: "お名前を聞き取れるまで"
  saves:
    - 氏名:
        type: string
        description: "お客様の氏名"
- 指示: "{{ 氏名 }}様、ありがとうございます"
```

## 終了

通話を終了します。直前の **指示** の発話が完了した後に通話が切断されます。

```yaml
- 指示: "受付完了の旨を伝え、終話の挨拶をする"
- 終了: true
```

## 実践例

氏名と電話番号をヒアリングするフローを、ステップごとに見ていきましょう。

<Steps>
  <Step title="base_prompt でAIの役割を定義">
    ```yaml
    base_prompt: |
      あなたはカスタマーサポートのAIオペレーターです。
      丁寧な対応を心がけてください。
    ```
  </Step>

  <Step title="名前をヒアリング">
    **指示** で質問し、**待機** + saves で回答を待ちます。
    ```yaml
    - 指示: "お名前をフルネームで教えてください。聞き取ったら復唱して確認します"
    - 待機: "ユーザーが復唱確認に同意するまで"
      saves:
        - 氏名:
            type: string
            description: "お客様の氏名"
    ```
  </Step>

  <Step title="電話番号をヒアリング">
    同じ **指示** → **待機** パターンで次の情報を収集します。
    ```yaml
    - 指示: "{{ 氏名 }}様、ありがとうございます。次に連絡先の電話番号を教えてください"
    - 待機: "電話番号を聞き取り、復唱確認に同意するまで"
      saves:
        - 電話番号:
            type: string
            description: "連絡先電話番号"
    ```
  </Step>

  <Step title="終話">
    情報収集が完了したら、挨拶して通話を終了します。
    ```yaml
    - 指示: "受付が完了しました。ご利用ありがとうございました"
    - 終了: true
    ```
  </Step>
</Steps>

### 完成形

```yaml
base_prompt: |
  あなたはカスタマーサポートのAIオペレーターです。
  丁寧な対応を心がけてください。

steps:
  - 指示: "お名前をフルネームで教えてください。聞き取ったら復唱して確認します"
  - 待機: "ユーザーが復唱確認に同意するまで"
    saves:
      - 氏名:
          type: string
          description: "お客様の氏名"
  - 指示: "{{ 氏名 }}様、ありがとうございます。次に連絡先の電話番号を教えてください"
  - 待機: "電話番号を聞き取り、復唱確認に同意するまで"
    saves:
      - 電話番号:
          type: string
          description: "連絡先電話番号"
  - 指示: "受付が完了しました。ご利用ありがとうございました"
  - 終了: true
```

## よくある待機パターン

| パターン | 例 |
|---------|-----|
| 復唱確認 | `"ユーザーが復唱確認に同意するまで"` |
| 情報の聞き取り | `"お名前を聞き取れるまで"` |
| 回答待ち | `"ユーザーが回答するまで"` |
| 不明時スキップ | `"住所を聞き取るか、わからないと回答するまで"` |
| Yes/No確認 | `"ユーザーが了承するまで"` |

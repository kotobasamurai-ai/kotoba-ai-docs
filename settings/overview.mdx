エージェント設定画面の「基本設定」タブで設定できる全項目のリファレンスです。

## エージェント名

ダッシュボードのエージェント一覧やセッション履歴に表示される名前です。用途がわかりやすい名前を付けることを推奨します。

**例**: `受付対応エージェント`、`予約確認エージェント`

## ベースプロンプト

AIの役割・基本方針を定義するシステムプロンプトです。Kotobaフローの `base_prompt` と同じ内容が設定されます。

ベースプロンプトには以下を含めることを推奨します：

- **AIの役割**（例: カスタマーサポート担当）
- **話し方のトーン**（例: 丁寧語、フレンドリー）
- **対応範囲**（例: 製品に関する質問のみ対応）
- **禁止事項**（例: 個人情報の開示禁止）

```yaml
base_prompt: |
  あなたは株式会社〇〇のカスタマーサポート担当AIです。
  丁寧な敬語で対応してください。
  製品に関する質問にのみ回答し、それ以外の質問には対応できない旨を伝えてください。
```

<Tip>
  ベースプロンプトは具体的に書くほど、AIの応答が安定します。「丁寧に対応して」よりも「です・ます調の敬語で、相手の名前を呼びながら対応して」のように具体的に記述しましょう。
</Tip>

## 言語モデル

エージェントが使用する言語モデル（LLM）を選択します。モデルによってレイテンシ（応答速度）と精度が異なります。

| モデル | レイテンシ | 特徴 |
|--------|-----------|------|
| Gemini 3 Flash | ~1.0秒 | 高精度。複雑な会話フローに推奨 |
| Gemini 2.5 Flash | ~0.5秒 | 標準精度。応答速度を重視する場合に推奨 |
| GPT-4.1 | ~0.6秒 | 標準精度。OpenAI モデル |

<Info>
  レイテンシはネットワーク環境や会話の複雑さによって変動します。上記は目安です。
</Info>

## 音声設定（TTS）

テキスト読み上げ（Text-to-Speech）の音声パラメータを設定します。

| パラメータ | 範囲 | デフォルト | 説明 |
|-----------|------|-----------|------|
| 音声 | 女性 / 男性 | 女性 | エージェントの音声の種類 |
| 話速 | 0.5〜2.0 | 1.0 | 発話のスピード。1.0 はやや速めなので、遅くしたい場合は値を下げてください |
| 音量補正 | -10〜+10 dB | 0 | 音量の調整。プラスで大きく、マイナスで小さく |

<Tip>
  **話速の目安**: デフォルトの 1.0 はやや速く感じることがあります。ゆっくり話させたい場合は 0.8〜0.9 に下げるのがおすすめです。
</Tip>

## VAD設定（音声区間検出）

VAD（Voice Activity Detection）は、ユーザーが話しているかどうかをリアルタイムで判定する機能です。VADの設定を調整することで、「AIが早く割り込みすぎる」「ユーザーの発話が途中で切れる」といった問題を改善できます。

| パラメータ | 範囲 | デフォルト | 説明 |
|-----------|------|-----------|------|
| 感度 | 1.0〜15.0 | 6.0 | 音声検出の感度。高いほど小さな音でも音声と判定 |
| 音声保持時間 | 0.0〜1.0秒 | 0.1秒 | 音声が途切れた後も「発話中」とみなす時間 |
| 最小音声長 | 0.0〜0.5秒 | 0.2秒 | この長さ未満の音声は割り込みとして検知しない |

<Tip>
  **AIが早く割り込む場合**: 音声保持時間を長くしてください（例: 0.1 → 0.3）。ユーザーが言葉を選んでいる間の短い沈黙を「発話終了」と判定しにくくなります。

  **雑音で誤反応する場合**: 感度を下げてください（例: 6.0 → 4.0）。また、最小音声長を長くする（例: 0.2 → 0.3）ことで、短いノイズを無視できます。

  **ユーザーの発話を検出できない場合**: 感度を上げてください（例: 6.0 → 8.0）。電話回線の音質が悪い場合に有効です。
</Tip>

## デフォルト転送設定

Kotobaフローの `転送` コマンドで使用されるデフォルト値を設定します。フロー側で個別に値を指定した場合は、そちらが優先されます。

### 転送タイプ

| タイプ | 説明 |
|--------|------|
| SIP転送（Warm） | AIが転送先オペレーターに引き継ぎメッセージを伝えてから、通話者を接続します |
| 外線転送（Cold） | 通話者を直接転送先に接続します。引き継ぎメッセージは使用されません |

### 転送パラメータ

| パラメータ | 説明 |
|-----------|------|
| 転送先 | SIP URI（例: `sip:operator@example.com`）または電話番号（例: `+81312345678`） |
| 引き継ぎメッセージ | Warm転送時に、転送先オペレーターに伝えるメッセージ。会話の要約や顧客情報を含めることができます |
| 応答待ちタイムアウト | 転送先が応答するまでの待機時間（秒）。タイムアウトした場合、通話者に転送できなかった旨を伝えます |

<Tip>
  引き継ぎメッセージには `{{ 変数名 }}` でKotobaフローの変数を埋め込めます。例: `「{{ 氏名 }}様からのお電話です。{{ 問い合わせ内容 }}についてのお問い合わせです」`
</Tip>

### Kotobaフローでの転送コマンド

フロー内で `転送` コマンドを使うと、ここで設定したデフォルト値で転送が実行されます。

```yaml
- 指示: "担当者におつなぎしますので、少々お待ちください"
- 転送: true
```

フロー側で個別に設定を上書きすることもできます。詳しくは[フロー制御](/agents/flow-control)を参照してください。

## 無応答タイムアウト

ユーザーが一定時間応答しない場合の動作を設定します。

| パラメータ | デフォルト | 説明 |
|-----------|-----------|------|
| 待機秒数 | 15秒 | ユーザーの応答を待つ時間 |
| 確認メッセージ | — | 無応答時に順番に再生されるメッセージ（複数設定可） |
| 終話メッセージ | — | すべての確認メッセージ再生後、通話を終了する前に再生されるメッセージ |

### 動作の流れ

1. ユーザーが **待機秒数** の間応答しない
2. **確認メッセージ**（1つ目）を再生し、再び待機
3. まだ応答がない場合、**確認メッセージ**（2つ目）を再生し、再び待機
4. すべての確認メッセージを再生しても応答がない場合、**終話メッセージ** を再生して通話を終了

<Info>
  確認メッセージは複数設定でき、設定した順番に再生されます。例えば「お客様、聞こえていますでしょうか？」→「お声が聞こえないようですので…」のように段階的に案内できます。
</Info>

## 会話圧縮設定

長い通話ではAIに送信するメッセージ数が増え、言語モデルのトークン制限に近づきます。会話圧縮は、一定のメッセージ数を超えた時点で過去の会話を自動的に要約し、トークン使用量を抑える機能です。

| パラメータ | デフォルト | 説明 |
|-----------|-----------|------|
| 圧縮しきい値 | 10メッセージ | この数を超えると過去の会話が要約されます |
| 圧縮用プロンプト | — | 要約時に使用するカスタムプロンプト |

### 仕組み

1. 会話のメッセージ数が **圧縮しきい値** を超える
2. 過去のメッセージが **圧縮用プロンプト** に従って要約される
3. 要約されたテキストが新しいコンテキストとして使用される
4. 以降の会話は要約 + 最新メッセージで続行

<Tip>
  圧縮用プロンプトをカスタマイズすると、要約の質を調整できます。例えば「顧客名と問い合わせ内容は必ず保持すること」のように、重要な情報の保持を指示できます。
</Tip>

<Info>
  短い通話（数分程度）では圧縮が発生することはほとんどありません。長時間の問い合わせ対応やヒアリングが多い場合に効果的です。
</Info>

## 通話後ツール

通話終了後に自動的に実行されるツールを設定します。通話内容の記録や外部システムへのデータ連携に使用します。

### 設定方法

1. エージェント設定画面の「基本設定」タブを開く
2. 「通話後ツール」セクションまでスクロール
3. 使用したいツールのトグルをONにする
4. 必要に応じてパラメータを設定

<Info>
  通話後ツールとして利用できるのは、あらかじめ[ツール管理ページ](/agents/tools)で設定済みのツールです。新しいツールを追加するには、まずツール管理ページで連携設定を行ってください。
</Info>

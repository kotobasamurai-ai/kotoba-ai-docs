ルーティングは、着信先の電話番号やSIP宛先を、どのエージェントに接続するかを決める設定です。エージェントを作成しただけでは電話は受けられず、実際に通話を流すにはルーティング設定が必要です。

**設定例**: `03-1234-5678` → `受付エージェント v3`

## 一覧画面の見方

画面上部では、ルーティングを次のタブで絞り込めます。

| タブ | 説明 |
| --- | --- |
| すべて | すべてのルーティングを表示します |
| 電話番号 | 電話番号ルーティングのみ表示します |
| SIP | SIPルーティングのみ表示します |

右上の「追加」から新しいルーティングを作成します。

一覧カードでは次の情報を確認できます。

| 項目 | 説明 |
| --- | --- |
| 着信先 | 電話番号またはSIP宛先 |
| 割り当て先 | エージェント |
| 使用バージョン | エージェントに割り当てた公開版 |
| ステータス | アクティブか非アクティブか |

## 新規作成の流れ

1. 「追加」をクリックする
2. チャネル種別を選ぶ
3. 着信先を指定する
4. 割り当て先のエージェントを選ぶ
5. 使用する公開バージョンを選ぶ
6. アクティブ状態を確認して保存する

## チャネル種別

ルーティングには2種類あります。

### 電話番号

組織に登録済みの電話番号から選択します。

向いているケース:

- 代表電話
- サポート窓口
- 予約受付番号
- キャンペーン専用番号

### SIP

SIP識別子または電話番号を手入力します。

入力例:

- `2a8245ae`
- `+815012345678`

SIPドメインが設定されている場合、画面上では `sip:識別子@ドメイン` の形式で表示されます。

向いているケース:

- SIPトランク連携
- PBXや外部音声基盤からの着信
- システム間接続

<Tip>
  SIPルーティングでは、外部PBXやSIP基盤が実際にどの識別子で着信させるかを先に確認しておくと設定ミスを防げます。
</Tip>

## 割り当て先

着信させるエージェントを選び、次の2つを指定します。

- エージェント名
- バージョン

ルーティングは「どのエージェントに接続するか」だけでなく、「どの公開バージョンを使うか」まで固定します。

同じエージェントでも、次のように分けて運用できます。

- `代表受付` の `v2` を使うルーティング
- `代表受付` の `v3` を使うルーティング

## エージェントの公開が必要

選んだエージェントに公開バージョンが1つもない場合、ルーティングを作成できません。表示される確認ダイアログから先に公開を行ってください。

<Info>
  ルーティング前に、必ずエージェントを公開してください。着信時に使用されるのは公開バージョンのみです。
</Info>

## アクティブ設定

各ルーティングにはアクティブのON/OFFがあります。

| 状態 | 説明 |
| --- | --- |
| アクティブ | その着信先で通話を受けられる状態です |
| 非アクティブ | 設定は残りますが、利用できない状態です |

一時停止したい場合は、削除せず、まず非アクティブにする運用を推奨します。

## 編集できること

既存ルーティングでは、主に次の変更ができます。

- 割り当て先を別のエージェントに変える
- 使用バージョンを切り替える
- アクティブ状態を切り替える

ただし、チャネルそのものは編集画面では固定です。既存の電話番号やSIP識別子自体を変更したい場合は、作り直してください。

## 削除

ルーティングを削除すると、その着信先はエージェントにつながらなくなります。削除は取り消せません。

<Warning>
  一時的に停止する場合は、削除ではなく非アクティブに設定してください。
</Warning>

## 設定例

### 電話番号をエージェントへ

| 項目 | 設定例 |
| --- | --- |
| チャネル種別 | 電話番号 |
| 電話番号 | `03-1234-5678` |
| 割り当て先 | エージェント |
| エージェント | `代表受付` |
| バージョン | `v3` |
| アクティブ | ON |

### SIP着信をエージェントへ

| 項目 | 設定例 |
| --- | --- |
| チャネル種別 | SIP |
| SIP識別子 | `2a8245ae` |
| 割り当て先 | エージェント |
| エージェント | `サポート受付` |
| バージョン | `v1` |
| アクティブ | ON |
